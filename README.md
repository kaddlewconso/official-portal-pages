# official-portal-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面可作为门户或入口页面使用，适用于各类展示、导航或信息呈现场景。仓库本身不绑定特定域名或网站，页面内容具有通用性和可移植性。

## 目录说明

```
official-portal-pages/
├── pages/          # 存放各独立 HTML 页面文件
├── assets/         # 页面引用的静态资源（如 CSS、JS、图片）
├── templates/      # 页面模板或布局文件（可选）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面为一个独立文件，命名清晰，便于识别用途。
- **assets/**：按需存放页面所需的样式、脚本及媒体资源，建议按类别组织子目录。
- **templates/**：若存在通用页面结构或布局模板，可放置于此目录。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务，可直接在浏览器中运行。
- 页面之间相互独立，无耦合关系，可单独部署或引用。
- 归档时建议保留原始文件结构，以维护资源引用路径的有效性。
- 若页面包含外部依赖（如 CDN 库），请在页面内明确标注来源。

## 维护说明

- 欢迎提交新页面或对现有页面进行改进，请遵循以下原则：
  - 保持页面独立性与轻量化。
  - 避免硬编码具体域名或第三方链接。
  - 确保页面在不同屏幕尺寸下具有基本的可用性。
- 提交前请检查文件路径和资源引用是否正确。
- 如需删除或归档旧页面，建议先移入 `archived/` 目录，避免直接删除历史记录。

## 使用建议

- 可直接将 `pages/` 目录下的 HTML 文件部署至任意静态托管服务。
- 也可通过 Git Pages 或类似服务将整个仓库作为静态站点发布。
- 页面内容可根据实际需求自由修改，无需额外授权。
